# stock-app

## プロジェクト概要

株式・投資情報を管理・表示するWebアプリケーション。

## 技術スタック

### フロントエンド
- **React** 18.2.0 — UIフレームワーク
- **Vite** 5.0.8 — ビルドツール・開発サーバー
- **Recharts** 2.10.0 — チャート描画ライブラリ

### バックエンド（データ取得）
- **Google Apps Script（GAS）** — Yahoo Finance からの株価データ取得（日本株・米国株対応）
- **Alpha Vantage API** — 米国株データ取得（フォールバック用、無料プラン25回/日）

- **Yahoo Finance** `query1.finance.yahoo.com/v8/finance/chart/` — 株価履歴データ
- **Yahoo Finance Japan** `query.finance.yahoo.co.jp/v8/finance/chart/` — 日本株の会社名取得

### インフラ・デプロイ
- **GitHub Pages** — 静的サイトホスティング（公開URL: https://nori1975461.github.io/stock-app）

### 予測ロジック概要（CLEAR TRADE理論）
- **MA/RSI/MACD/ボリンジャー等の従来指標は一切使用しない**
- 出来高＋純粋チャート分析の9指標でスコアリング
- stableScore（7指標・週次安定）でランキング、score（9指標）で詳細表示
- CT理論の4次元構造をすべて実装済み（次元1〜4）

## 主要機能

- **次元1：相場環境判断** — 米国3＋日本13セクター代表銘柄をスキャンし EASY/NORMAL/HARD を判定
- **次元2：先導株分析** — CT_LEADERS 8銘柄のリアルタイムCT分析（モナリザ理論）
- **次元3：CT銘柄スクリーニング** — CT_UNIVERSE 60銘柄を一括スキャンして上位10銘柄を抽出
- **次元4：エントリータイミング** — 詳細ビューに初速判定・2日目確認を表示
- 単一銘柄の詳細分析（CT9指標・分散エグジット判断・1か月後見通し・チャート）
- 複数銘柄（最大**10**銘柄）の上昇予測ランキング＋購入推奨カード
- 分析期間：1週間〜12週間（6段階）
- GAS URL・Alpha Vantage APIキーを localStorage に保存（再入力不要）
- 日本株（.T）・米国株の両対応

## 既知の制限・注意事項

- 日本語会社名はGASの静的マッピング（約130銘柄）で対応済み。未収録銘柄は英語名表示
- Alpha Vantage 無料プランは25回/日の上限あり（複数銘柄分析で消費が早い）
- Yahoo Finance は非公式APIのため、予告なく仕様変更・遮断の可能性あり
- 予測ロジックはCT理論（出来高＋チャート）のみ（ファンダメンタルズ・ニュース等は未考慮）
- 投資判断の根拠としての利用は不可（参考情報のみ）

## Git 運用ルール

**コードを変更するたびに必ずGitHubへプッシュすること。**

```bash
git add <変更ファイル>
git commit -m "変更内容を簡潔に記述"
git push origin main
```

### 基本方針

- コミットは意味のある単位で行う（機能単位、バグ修正単位など）
- コミットメッセージは日本語でも英語でも可。変更の「なぜ」を明記する
- `main` ブランチへ直接プッシュする（小規模プロジェクトのため）
- 破壊的な変更（ファイル削除、大規模リファクタ）は事前にユーザーへ確認する
- `--no-verify` や `--force` は使用しない（明示的な指示がある場合を除く）

### コミットの流れ

1. `git status` で変更ファイルを確認
2. 関係するファイルを個別に `git add` でステージング（`git add -A` は避ける）
3. `git commit -m "..."` でコミット
4. `git push origin main` でGitHubへプッシュ

## CLEAR TRADE理論（CT理論）実装原則

**このアプリはCT理論（Mr.K）に基づく。MA/RSI/MACD/ボリンジャー等の従来指標はスコアに使用しない。**

新しいCT指標を実装・改善する際は、必ず以下の3点を確認すること：

1. **この指標は何を測っているか**
   - 「銘柄の質」= 中長期トレンドの強さ・継続性 → スクリーニングスコアに含める
   - 「エントリータイミング」= 今この瞬間に入るかどうかの確認 → 詳細ビューのみ

2. **この指標の時間軸はどこか**
   - 週次評価（複数日〜数週間で形成）→ スクリーニングスコアに含めてよい
   - 日次・一度限り（エントリー前の瞬間判断）→ スクリーニングスコアに含めてはならない

3. **スクリーニングスコアに含めるべきか、詳細ビューに留めるべきか**
   - スクリーニングスコア（`stableScore`）= 銘柄の質指標のみ。ランキング・先導株ソートに使用
   - 詳細ビュー = エントリータイミング指標（2日目確認・初速判定）を「エントリータイミング」セクションとして表示

**この判断を怠ると、エントリータイミング指標が毎日再計算されてランキングが日次で大きく変動する。**
これはCT理論の「銘柄スクリーニングは週次で行う」という原則に反する。

CT理論の2段階構造：
```
Stage 1：銘柄スクリーニング（週次）
  出来高3日平均・OBV15日・規律可能性15日・マグネット効果90日
  停滞シグナル・連続日数・ローソク足10日 → stableScore

Stage 2：エントリータイミング確認（日次・一度限り）
  初速判定・2日目確認 → 詳細ビュー専用
```

## 開発ガイドライン

- セキュリティ脆弱性（XSS、SQLインジェクション等）を絶対に混入しない
- 不要なコメントは書かない（「なぜ」が非自明な場合のみコメントを追加）
- 既存ファイルの編集を優先し、不要な新規ファイルは作成しない
- タスクに必要な最小限の変更のみを行う（過剰な抽象化・リファクタは不要）
